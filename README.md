# Тренажёр тайского алфавита (рукописное письмо)

PWA-тренажёр для **написания от руки и запоминания тайского алфавита**. Рассчитан на телефон со стилусом (например, Samsung Galaxy S25 Ultra + S-Pen): пишешь прямо по экрану, нажатие пера управляет толщиной линии, касания ладонью игнорируются.

## Возможности

- **Обводка по шаблону** — буква показана бледным контуром (из шрифта), обводишь стилусом сверху; есть подсказка «покрытие контура» в %.
- **Анимация порядка штрихов** — показывает, как ведётся линия (нумерованные точки старта и стрелки).
- **Письмо по памяти + самооценка** — по подсказке (транскрипция/звук) пишешь букву, открываешь эталон, оцениваешь себя.
- **Интервальное повторение (SRS)** — алгоритм SM-2 сам подбирает карточки «к повторению»; прогресс сохраняется локально и работает офлайн.
- **Озвучка** названия буквы через системный голос (Web Speech API, при наличии тайского TTS).
- **Контент**: 44 согласных (с классом тона) и основные гласные.

## Запуск локально

ES-модули и шрифт требуют HTTP (не `file://`):

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Деплой (GitHub Pages)

Settings → Pages → Build and deployment → Source: *Deploy from a branch*, выбрать ветку и папку `/ (root)`. После сборки открыть выданный URL на телефоне и «Добавить на домашний экран» — приложение установится как PWA и будет работать офлайн.

## Структура

```
index.html                 каркас + регистрация service worker
css/styles.css             тёмная тема, крупные элементы
js/app.js                  состояние, экраны, склейка модулей
js/template.js             извлечение контура глифа из шрифта (opentype.js)
js/pointer.js              рукописный ввод: нажатие→толщина, palm-rejection, отмена/очистка
js/animate.js              анимация порядка штрихов
js/srs.js                  планировщик SM-2 (localStorage)
js/audio.js                озвучка (Web Speech API)
js/data/consonants.js      44 согласных + метаданные
js/data/vowels.js          основные гласные + метаданные
lib/opentype.min.js        парсер шрифта (SIL OFL / MIT)
fonts/                     Noto Sans Thai Looped (SIL OFL 1.1)
sw.js, manifest.webmanifest, icons/   PWA
```

## Порядок штрихов (как улучшать)

Готового открытого датасета порядка штрихов для тайского нет, поэтому по умолчанию анимация обводит контур глифа из шрифта. Чтобы задать **правильный учебный порядок**, заполните поле `strokes` у буквы в `js/data/*.js` — массив «осевых линий» в нормализованных координатах `[0..1]`, в порядке написания:

```js
{ char: "ก", /* ... */, strokes: [
  { points: [[0.30,0.25],[0.32,0.40], /* ... */] },  // штрих 1 (с «головы»)
  { points: [[0.55,0.20],[0.55,0.78]] }              // штрих 2
] }
```

Если `strokes` непустой — анимация и стрелки направления строятся по нему. Координаты сверяйте с авторитетными источниками по тайскому письму.
